# MACUIN — Guía de desarrollo

## Estándares de código

- Python: `snake_case` para variables, funciones y archivos. Clases en `PascalCase`.
- PHP/Laravel: convenciones estándar de Laravel (`camelCase` para métodos, `PascalCase` para clases).
- No agregar comentarios innecesarios. Solo comentar lógica no obvia.
- Mantener los nombres y estructura de archivos existentes; no renombrar sin motivo.
- No añadir docstrings, type hints ni manejo de errores especulativos a código que no se está modificando.
